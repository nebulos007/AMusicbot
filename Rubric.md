# AI Chatbot Capstone Project Rubric
Students will design and build a functional chatbot that integrates with external data sources, uses AI orchestration libraries, and provides intelligent responses based on retrieved information. You will need to turn in a Git repository containing your chatbot code, your documentation in markdown, and a short video demonstration of your chatbot running with a walkthrough of your solution.

## Scoring Scale: 4 (Excellent) | 3 (Proficient) | 2 (Developing) | 1 (Beginning)

### 1. Data Source Integration (25 points)
Function Calling Implementation (15 points)
Excellent (13-15 pts): Implements multiple, diverse data sources with robust error handling. Functions are well-designed, properly documented, and handle edge cases gracefully. Demonstrates sophisticated understanding of API integration patterns.
Proficient (10-12 pts): Successfully integrates 2+ data sources with basic error handling. Functions work reliably for standard use cases with clear documentation.
Developing (7-9 pts): Integrates 1-2 data sources with minimal error handling. Functions work but may have reliability issues or limited documentation.
Beginning (1-6 pts): Limited or non-functional data source integration. Functions may not work consistently or lack proper implementation.
Data Quality & Relevance (10 points)
Excellent (9-10 pts): Data sources are highly relevant to chatbot purpose. Demonstrates thoughtful curation and preprocessing of data for optimal AI responses.
Proficient (7-8 pts): Data sources are relevant and appropriately chosen for the application domain.
Developing (5-6 pts): Data sources are somewhat relevant but may not fully align with chatbot's stated purpose.
Beginning (1-4 pts): Data sources are poorly chosen or irrelevant to the application.
### 2. AI Orchestration (25 points)
Library Usage & Implementation (15 points)
Excellent (13-15 pts): Sophisticated use of orchestration library (LangChain, LlamaIndex, etc.) with custom components, proper chain management, and optimized workflows. Demonstrates deep understanding of library capabilities.
Proficient (10-12 pts): Competent use of orchestration library with standard patterns and configurations. Shows solid understanding of core concepts.
Developing (7-9 pts): Basic implementation of orchestration library with minimal customization. May rely heavily on examples without deeper understanding.
Beginning (1-6 pts): Poor or incorrect use of orchestration tools. May attempt to build everything from scratch without leveraging library capabilities.
Conversation Management (10 points)
Excellent (9-10 pts): Maintains coherent, contextual conversations across multiple turns. Handles complex conversation flows and state management elegantly.
Proficient (7-8 pts): Maintains basic conversation context and handles multi-turn interactions adequately.
Developing (5-6 pts): Limited conversation management with occasional context loss or confusion.
Beginning (1-4 pts): Poor conversation flow with frequent context issues or inability to maintain coherent dialogue.
### 3. Response Quality & Data Utilization (20 points)
Answer Accuracy & Relevance (12 points)
Excellent (11-12 pts): Consistently provides accurate, relevant answers that effectively synthesize information from multiple data sources. Shows excellent prompt engineering for response quality.
Proficient (9-10 pts): Generally provides accurate, relevant answers using retrieved data effectively.
Developing (6-8 pts): Answers are sometimes accurate but may miss context or fail to fully utilize available data.
Beginning (1-5 pts): Frequently provides inaccurate or irrelevant responses, poor utilization of data sources.
Information Synthesis (8 points)
Excellent (7-8 pts): Skillfully combines information from multiple sources into coherent, comprehensive responses. Handles conflicting information appropriately.
Proficient (6 pts): Adequately combines information from different sources with clear attribution.
Developing (4-5 pts): Basic combination of information sources with some success.
Beginning (1-3 pts): Poor integration of multiple information sources.
### 4. Technical Implementation (15 points)
Code Quality & Architecture (8 points)
Excellent (7-8 pts): Clean, well-organized code with proper separation of concerns, following best practices for the chosen programming language. Demonstrates good software engineering principles.
Proficient (6 pts): Generally well-structured code that is readable and maintainable.
Developing (4-5 pts): Code works but may have organizational or style issues.
Beginning (1-3 pts): Poor code quality with significant structural problems.
Error Handling & Robustness (7 points)
Excellent (6-7 pts): Comprehensive error handling for API failures, network issues, and edge cases. Graceful degradation when services are unavailable.
Proficient (5 pts): Basic error handling that prevents crashes and provides user feedback.
Developing (3-4 pts): Limited error handling with some unhandled edge cases.
Beginning (1-2 pts): Poor error handling that results in crashes or unclear error states.
### 5. User Experience & Interface (10 points)
Interface Design (5 points)
Excellent (5 pts): Intuitive, polished interface that enhances user interaction. May include advanced features like typing indicators, message formatting, or media support.
Proficient (4 pts): Clean, functional interface that serves the application's needs well.
Developing (3 pts): Basic but usable interface with minor usability issues.
Beginning (1-2 pts): Poor interface design that hinders user interaction.
User Feedback & Guidance (5 points)
Excellent (5 pts): Provides clear guidance on capabilities, helpful error messages, and intuitive user onboarding.
Proficient (4 pts): Adequate user guidance and feedback systems.
Developing (3 pts): Basic user feedback with some gaps in guidance.
Beginning (1-2 pts): Little to no user guidance or feedback.
### 6. Documentation & Presentation (5 points)
Project Documentation (3 points)
Excellent (3 pts): Comprehensive documentation including setup instructions, architecture overview, API documentation, and usage examples.
Proficient (2 pts): Good documentation covering key aspects of the project.
Developing (1-2 pts): Basic documentation with some missing elements.
Beginning (0-1 pts): Poor or missing documentation.
Final Presentation (2 points)
Excellent (2 pts): Clear, engaging presentation demonstrating all requirements with live demo and thoughtful discussion of design decisions.
Proficient (1-2 pts): Solid presentation covering project requirements with demonstration.
Developing (1 pt): Basic presentation with limited demonstration or explanation.
Beginning (0 pts): Poor presentation that fails to adequately demonstrate the project.
